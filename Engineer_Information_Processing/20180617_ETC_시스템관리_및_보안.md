## 정보 보안 및 암호화  

#### 보안의 정의와 보안 요건  
* 보안의 정의  
  * 컴퓨터 시스템 및 컴퓨터에 저장된 정보들을 일부분 또는 전체에 대해 권한이 없는 **외부의 불법적인 침입이나 변경,파괴로부터 보호하는 것**  
* 보안 요건  
  * **기밀성,무결성,가용성,인증,부인방지,접근통제** 로 이뤄짐  
  * **기밀성**  
    * 시스템 내의 정보와 자원은 인가된 사용자에게만 접근이 허용  
    * 정보가 전송중에 노출되더라도 데이터를 읽을 수 없음  
  * **무결성**  
    * 시스템 내의 정보는 인가된 사용자만 수정이 가능  
    * 정보의 내용이 전송 중에 수정되지않고 전달되는 것을 의미  
  * **가용성**  
    * 인가받은 사용자는 언제라도 사용 가능함  
  * **인증**  
    * 정보를 보내오는 사람의 신원을 확인함  
    * 사용자를 식별하고, 사용자의 접근 권한을 검증  
  * **부인 방지**  
    * 데이터를 송,수신한 자가 송,수신한 사실을 부인할 수 없도록 송,수신 증거를 제공함  
  * **접근 통제**  
    * 시스템의 자원 이용에 대한 불법적인 접근을 방지하는 과정을 말하며 크래커의 침입으로부터 보호  

#### 보안 위협의 유형  
* 가로막기,가로채기,수정,위조가 있다  
  * 가로막기  
    * 데이터의 흐름을 방해하는 행위  
    * 가용성을 저해  
  * 가로채기  
    * 중간에 데이터를 도청하여 정보를 유출하는 행위  
    * 기밀성을 저해  
  * 수정  
    * 원래의 데이터를 다른 내용으로 바꾸는 행위  
    * 무결성을 저해  
  * 위조  
    * 송신자를 속여, 데이터를 송신하는 행위  
    * 무결성을 저해  

#### 인증  
* 다중 사용자 컴퓨터 시스템 또는 망 운용 시스템에서 시스템이 단말 작동 개시 정보를 확인하는 보안 절차  
* 망을 경유해서 컴퓨터에 접속해 오는 사용자가 등록된 사용자인지 확인하는것과  
전송된 메세지가 변조되거나 의미가 그릇되지않고 송신자가 보낸 그대로의 것인지를 확인하는 것으로 구분  

#### 개인키 암호화 기법  
* 정의  
  * 동일한 키로 데이터를 암호화하고 복호화하는 암호화기법  
  * 단일키 암호화 기법 혹은 대칭키 암호화 기법이라함
  * 대표적으로 DES기법이 있음
* 장점
  * 암호/복호화 속도가 빠름
  * 알고리름이 단순
  * 파일의 크기가 공개키 암호 기법보다 적음
* 단점
  * 사용자가 증가하면 관리해야할 키도 많아짐
<br></br>
* 구조

#### 공개키 암호화 기법  
* 정의  
  * 데이터 암호화 시 공개키로 암호화하고, 복호화 시 비밀키로 복호화하는 암호화기법(비밀키는 데이터베이스 사용권한이 있는 사용자만 나눠가짐)  
  * 비대칭 암호 기법이라함  
  * 대표적으로 RSA기법이 있음
* 장점
  * 키의 분배가 용이하고 관리해야할 키의 갯수가 적음
* 단점
  * 암호/복호화 속도가 느림,
  * 알고리즘이 느림,
  * 대칭키 암호화기법보다 파일의 크기가 큼
<br></br>  
* 구조
![](https://www.ibm.com/support/knowledgecenter/ko/SSFKSJ_7.1.0/com.ibm.mq.doc/sy10500b.gif)

#### DES  
* 데이터 암호화 표준, 대표적인 대칭키 암호화 기법  
* 3개의 키를 연속해 놓은 트리플 DES가 많이 쓰임  

#### 순차적 암호화
* DB에서 암호화된 수치 데이터들이 원본 수치데이터와 동일한 순서로 정렬될 수 있도록 해주는 암호화 기술
* 검색속도 저하를 극복하는 암호화 방법
* 순서 자체가 중요한 정보가 될수 있어서 정보 노출의 위험성 존재

#### 공개키 기반 구조
* 공개키 암호 시스템을 안전하게 사용하고 관리하기 위한 정보 보호 표준 방식
* 대표적으로 ITU-T의 X.509와 비X.509 방식으로 구분
